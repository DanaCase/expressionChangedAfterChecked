What is expressionChangedAfterChecked?

It's an error that happens when something has changed in a compoent that's not reflected in the DOM.

AKA it's an error that happens because component changes happen outside of the normal Anguar change detection and rendering cycle.

This almost always happens because a child component changes something on the parent component

Lets generate some bad components

Contrived Component

In this contrived example, the child componet directly edits the parent property 'name' after the parent has rendered, but before the child has rendered

Shared Service Component

In this example, the parent and child component share a service. The parent has a listener that updates a property in response to changes in the service, and the child changes the service.

Circular Event Component

In this example, the parent listens to the child for an event, and the child executes the event in the ngOnInit callback

Bad Generation Sequence Example

In this example the parent component generates a child component after it should have rendered. Because the DOM is changed outside of the normal change cycle you get an expressionChangedAfterCHecked error
